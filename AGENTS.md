## Skill Loading

Before substantial work:

- Skill check: run `npx @tanstack/intent@latest list`, or use skills already listed in context.
- Skill guidance: if one local skill clearly matches the task, run `npx @tanstack/intent@latest load <package>#<skill>` and follow the returned `SKILL.md`.
- Monorepos: when working across packages, run the skill check from the workspace root and prefer the local skill for the package being changed.
- Multiple matches: prefer the most specific local skill for the package or concern you are changing; load additional skills only when the task spans multiple packages or concerns.

## Repository Shape

- Bun workspace monorepo; install with `bun install` and run commands from repo root unless a package script must run in that package.
- Workspaces are `interface`, `apps/*`, and `packages/*`; shared TS config is exported by `@recount/config` from `packages/config/src/base.tsconfig.json`.
- `apps/web` and `apps/desktop` are thin Vite/Tauri shells over `@recount/interface`; most React app code lives in `interface/src`.
- `apps/backend` serves health, HTTP API, Better Auth, and Effect RPC on `/rpc`; `apps/electric-proxy` serves Electric-facing HTTP endpoints. Both wire Effect layers in `src/index.ts`.
- Domain contracts/services live in `packages/core`; repository/infrastructure implementations live mostly in `packages/application`, `packages/db`, `packages/auth`, and `packages/notifications`.

## Commands

- `bun run dev` aliases `bun run dev:web`; it starts Caddy plus `@recount/backend`, `@recount/electric-proxy`, and `@recount/web` through Turbo.
- `bun run dev:desktop` swaps web for `@recount/desktop`; `bun run dev:all` starts web and desktop together.
- `bun run build` first runs `interface`'s `paraglide:compile` script, then `turbo build`.
- `bun run lint` is `turbo lint`; `bun run lint:ast` is `ast-grep scan`; `bun run typecheck` is `turbo typecheck`; `bun run format` is `oxfmt --write .`.
- The root `test` script intentionally fails as a guardrail; use Vitest directly, for example `bunx vitest run` or `bunx vitest run packages/db/test/database.layer.test.ts`. The interface package also has `bun run test` with `interface/vitest.config.ts`.
- Scope package checks with Turbo filters, for example `bunx turbo run typecheck --filter=@recount/interface`.
- `bun run clean` removes `node_modules` as well as Turbo outputs; do not run it as a routine build cleanup.
- `lefthook.yml` installs active pre-commit formatting/ast-grep hooks and pre-push lint/typecheck hooks.

## Generated Code And Config

- Do not hand-edit `interface/src/routeTree.gen.ts`; it is generated by `@tanstack/router-plugin/vite` from `interface/src/routes`.
- Paraglide output is generated under `interface/src/paraglide` by the interface Vite config and `interface` script `paraglide:compile`.
- Vite config is shared from `@recount/interface/vite`; web sets `__PLATFORM__` to `"web"` and port `8002`, desktop sets `__PLATFORM__` to `"desktop"` and port `1420`.
- `apps/desktop` `dev` and `build` run `prepare:sidecars` before Tauri/Vite; use `dev:vite` only when intentionally bypassing Tauri and the sidecar prep.
- `packages/ui` and `packages/editor` intentionally lint with `oxlint -A import/no-namespace .`; other packages use `oxlint .`.

## Local Services And Env

- Use `docker compose up -d` for local Postgres, Electric, and telemetry; Postgres is exposed on `localhost:5442`, Electric on `localhost:5133`, Grafana on `localhost:3000`, OTLP HTTP on `localhost:4318`.
- Backend and electric-proxy require `PORT`, `FRONTEND_ORIGINS`, and `DATABASE_URL`; auth also requires `BETTER_AUTH_SECRET`. Electric proxy routes also read `ELECTRIC_URL` plus optional `ELECTRIC_SOURCE_ID` and `ELECTRIC_SOURCE_SECRET`.
- The interface validates `VITE_BACKEND_URL` and `VITE_ELECTRIC_PROXY_URL` through `interface/src/lib/env.ts`.
- `packages/db` Drizzle commands use `packages/db/drizzle.config.ts`; `bun run db:push` in that package loads `.env.local` for `DATABASE_URL`.
- `interface` has its own Drizzle config for `interface/src/db/schema.ts` and migrations under `interface/src/db/migrations`.

## Code Conventions That Bite

- Root Oxlint extends Ultracite core and React presets; formatter is Oxfmt with 80-column width, double quotes, semicolons, generic array types (`Array<T>`), and sorted imports.
- Use top-level `import type` for type-only imports; `import/consistent-type-specifier-style` enforces this.
- Avoid non-null assertions in source code; config files sometimes disable the rule locally when reading required env.
- Use package export paths such as `@recount/core/modules/*` or `@recount/application/modules/*`; package-internal `#...` aliases are declared per package and are not portable across workspaces.
- In `interface`, do not import `~/domains/*/**`; import from the domain public `index.ts` such as `~/domains/workspace`.
- Effect code uses `Effect.gen`, `Layer`, typed services/errors, and `Config` for env; keep direct `process.env` access at config or test boundaries.
- `ast-grep` rules enforce Effect constraints such as no bare `new Error`, no manual JSON decode, no `throw`/`try-catch`/direct `fetch` inside Effect code, and no `console.log`.

## UI Work

- When editing `interface/` or `packages/ui/`, also follow that directory's `AGENTS.md`.
- Prefer existing Base UI-based components and exported `@recount/ui/*` entrypoints when editing shared UI.
- React Compiler is enabled through the shared interface Vite config, so do not add `useMemo`/`useCallback` by default unless existing local code or measurement justifies it.

## Verification

- For focused changes, run the nearest package `lint`/`typecheck` script or a filtered Turbo command first.
- For broad changes, run `bun run format`, `bun run lint`, `bun run lint:ast`, `bun run typecheck`, then `bunx vitest run`.
- Test ast-grep rules with `bun run lint:ast:test` after editing files under `ast-grep/`.
